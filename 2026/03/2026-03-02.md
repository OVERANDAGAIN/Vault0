下面我按“**模块之间怎么串起来**”来梳理（不是单点解释），并且把**每一步调用了谁、输入输出是什么形状**都标出来。你可以把它当成 OMG_AM 的“数据流图 + shape trace”。

---

## 0) 总览：OMG_AM 里两条主链路

OMG_AM 实际上只有两条“业务链”：

1. **推断链（online / rollout / policy 用）**
   `cal_subgoal_batched()`  或  `forward()`
   目标：用 **当前 obs + 历史条件** 推断 `subgoal z`

2. **训练链（omg_loss_func 用）**
   `omg_loss_func()` →（内部循环）→ `cvae.generate()` + `_cal_subgoal()`（VAE 监督）
   目标：训练 CVAE，让它的 posterior `(mu, logvar)` 贴近 VAE 给的“未来目标”分布，同时重构 `x`

---

## 1) 推断链 A：policy / world-model rollout 调用 `cal_subgoal_batched`

### Step A0：入口

**调用方（policy）**：
`subgoal = model_goal.cal_subgoal_batched(pre_actions, obs_last, obs_cur)`

**输入形状**

* `pre_actions`: `(B, N)` long
* `obs_last`: `(B, H, W, C)` 或 `(B, obs_dim)`
* `obs_cur`:  `(B, H, W, C)` 或 `(B, obs_dim)`

**输出形状**

* `subgoal`: `(B, D)`  （ego-only 后不再是 `(B,N,D)`）

---

### Step A1：obs_cur → CNN → x

在 `cal_subgoal_batched` 内部：

1. `_obs_to_chw(obs_cur)`

* 输入：`(B,H,W,C)` 或 `(B,obs_dim)`
* 输出：`cnn_inputs = (B,C,H,W)`

2. `shared_layers(cnn_inputs)`

* 输出：`(B, 32*H*W)`（因为最后 Flatten）

3. `flatten_layers(...)`

* 输出：`x = (B, x_dim)`（例如 32）

> **这条支路得到 CVAE 的“当前观测 embedding” `x`。**

---

### Step A2：构造 cond_inputs（历史条件）→ RNN → cond_x

同一个函数里继续：

1. `_obs_flat(obs_last)`

* 输入：`(B,H,W,C)` 或 `(B,obs_dim)`
* 输出：`last_obs_flat = (B, obs_dim)`

2. `F.one_hot(pre_actions, A)`

* 输入：`pre_actions (B,N)`
* 输出：`all_onehot = (B, N, A)`

3. 拆分动作（仍然需要 N 维）

* `self_action = all_onehot[:, ego, :]` → `(B, A)`
* `other = all_onehot; other[:, ego, :] = 0`
* `other_actions_flat = other.reshape(B, N*A)` → `(B, N*A)`

4. 拼 conditioning 向量

* `cond_inputs = cat([last_obs_flat, other_actions_flat, self_action], -1)`
* 形状：`(B, cond_dim)`
  其中 `cond_dim = obs_dim + N*A + A = obs_dim + (N+1)*A`

5. 给 RNN（加一个 time 维度=1）

* `cond_inputs = cond_inputs.unsqueeze(1)` → `(B, 1, cond_dim)`

6. `cond_rnn(cond_inputs, hidden_state)`

* 输入：

  * `inputs = (B,1,cond_dim)`
  * `hidden_state = (B, cond_emb_dim)`（ego-only hidden）
* 输出：

  * `cond_x_seq = (B,1,cond_emb_dim)`
  * `new_hidden = (B,cond_emb_dim)`（写回 self.hidden_state）

7. `cond_x = cond_x_seq.squeeze(1)` → `(B, cond_emb_dim)`

> **这条支路得到 CVAE 的条件 embedding `cond_x`，它把 “o_{t-1} + a_{t-1}” 压到一个向量里，并带着 GRU 的记忆。**

---

### Step A3：CVAE 推断 subgoal

最后：

`out, mu, log_var = cvae.forward_full(x, cond_x)`

* 输入：

  * `x: (B, x_dim)`
  * `cond_x: (B, cond_emb_dim)`
* 输出：

  * `out(z): (B, D)`
  * `mu/log_var: (B, D)`

**返回给 policy 的就是 `out(z)`**。

---

## 2) 推断链 B：EpisodeBatch 推断用 `forward(batch, t=None)`

这个是为了训练/缓存 `infer_mu, infer_log_var`，和原 OMG 框架兼容。

### Step B0：入口

`out = model_goal.forward(batch, t=None)`

**输出形状**

* `t is None`：`out = (bs, T, D)`
* `t provided`：`out = (bs, 1, D)`

### Step B1：它和 A 的区别是什么？

核心区别：`forward()` 不是直接喂 `obs_last, obs_cur, pre_actions`，而是：

* 从 `EpisodeBatch` 里取出：

  * `batch["obs"]` / `batch["state"]`
  * `batch["actions_onehot"]`
* 然后按时间 tt 构造：

  * `obs_cur(tt)`, `obs_last(tt)`
  * `last_actions_onehot(tt)`（把 actions_onehot 向右 shift 一步）
* 每个时间步都走一遍 **A1→A2→A3** 同样的数据流
* 最后把所有 `mu/log_var` 通过 `batch.holdup(...)` 写回去
  供 `omg_loss_func` 使用

> 换句话说：**forward 是“从 batch 自动生成 cal_subgoal_batched 的输入，然后循环跑 T 次”。**

---

## 3) 训练链：`omg_loss_func` 是怎么把 CVAE / VAE / eval_net 串起来的？

### Step C0：入口

`loss = model_goal.omg_loss_func(batch, eval_net, subgoal_mode=True)`

这里面同时调用：

* `cond_rnn`（RNN）
* `cvae.generate()`（训练重构）
* `_cal_subgoal()` → `vae.generate()` + `eval_net.compute_value()`（监督信号）

---

### Step C1：先 rollout 一遍 CVAE（重构 x + 得到 mu/logvar）

在 `omg_loss_func` 内部时间循环 `t=0..T-2`：

每个 t 都做：

1. 从 batch 取 ego 的 `obs_cur(t), obs_last(t)`
2. 从 batch 取 `last_actions_onehot(t)`（shift 后）
3. 构造 `cond_in = (bs,1,cond_dim)`（同推断链 A2）
4. `cnn_in -> x = (bs,x_dim)`（同推断链 A1）
5. `cond_rnn(cond_in, hidden)` → `cond_x=(bs,cond_emb_dim)`（同推断链 A2）
6. **关键：训练时用 `cvae.generate(x, cond_x)`，不是 forward_full**

   * 输出：

     * `x_in: (bs,x_dim)`（原 x）
     * `recons: (bs,x_dim)`（重构 x̂）
     * `mu/log_var/z: (bs,D)`

循环完 stack：

* `cvae_input: (bs, T-1, x_dim)`
* `cvae_output: (bs, T-1, x_dim)`
* `cvae_mu/log_var: (bs, T-1, D)`

---

### Step C2：再走 VAE “未来目标”链，得到 target (mu*, logvar*)

调用：
`subgoal, vae_mu, vae_log_var, target_obs = self._cal_subgoal(batch, eval_net, subgoal_mode, ...)`

这里 `_cal_subgoal` 本身又串了两块：

#### C2.1 未来 obs/state → VAE 编码

* 取 `obs_for_vae = ego_obs[:, 1:]`（未来候选，长度 T-1）
* flatten 成 `(bs*(T-1), obs_dim)`
* `vae.generate(obs_flat)` → `mu/log_var/z: (bs*(T-1), D)`
* reshape 回：

  * `subgoal_all = (bs, T-1, D)`
  * `mu_all = (bs, T-1, D)`
  * `log_var_all = (bs, T-1, D)`

#### C2.2 用 eval_net 选最优 k（horizon 内选未来哪一步作为“理想目标”）

* 构造候选：`subgoal_for_eval = (bs, T-1, horizon, D)`
  （本质是把 `subgoal_all` shift/pad 做 k=0..horizon-1）
* 构造 eval 的 obs：`obs_for_eval_h = (bs, T-1, horizon, obs_eval_dim)`
* 对每个 t 调：

  * `q = eval_net.compute_value(obs_for_eval_h[:,t], t_tensor, subgoal_for_eval[:,t])`
  * 得 `value(t) = (bs, horizon)`
* 拼起来 `value = (bs, T-1, horizon)`
* `k = argmax/argmin(value, dim=2)` → `(bs, T-1)`
* gather 回：

  * `final_mu/log_var/subgoal = (bs, T-1, D)`

这就是 `_cal_subgoal` 输出的 `vae_mu, vae_log_var`

---

### Step C3：最终 loss：reconstruction + KL 对齐

* `recons_loss = mse(cvae_output, cvae_input)`
  → `(bs, T-1)` 再 mask 后 mean
* `omg_kl = KL( q_cvae(mu,logvar) || q_target(mu*,logvar*) )`
  → `(bs, T-1)` 再 mask 后 mean
* `loss = recons_loss.mean() + alpha * omg_kl.mean()`

> 所以训练链完整闭环是：
> **CVAE 用 (x,cond_x) 推断 z 并重构 x**，同时用 **VAE(未来obs) + eval_net(k选择)** 提供 “目标分布”，逼近 CVAE posterior。

---

## 4) 你问的关键：为什么 ego-only 还会看到 N？

**因为 N 只存在于 conditioning 的“动作上下文”里：**

* `pre_actions: (B,N)`
* `all_onehot: (B,N,A)`
* `other_actions_flat: (B,N*A)`

但 **subgoal 输出已经不再有 N**：

* `cal_subgoal_batched`: `(B,D)`
* `forward`: `(bs,T,D)`

这和你最初那段结论是一致的：
原版同时服务 N 个 agent，所以输出 `(…,N,D)`；现在每个 policy 只服务 ego，因此只输出 ego 的 subgoal。
**输入中保留 N，是因为你仍然想让 ego 的 subgoal 受“所有人上一步动作”影响。**

## 5) 用一句话把模块联系起来（给你写在代码顶上的注释版）

* `shared_layers+flatten_layers`: `obs_cur -> x`（当前观测 embedding）
* `RNN(cond_rnn)`: `(obs_last + pre_actions) -> cond_x`（历史条件 embedding）
* `CVAE`: `(x, cond_x) -> z`（推断 subgoal latent）
* `VAE`: `future_obs -> z*`（把未来目标 obs 编码成 latent）
* `_cal_subgoal`: 用 `eval_net` 在 horizon 内挑 k，使得 `z*` 变成“最有价值的目标分布”
* `omg_loss_func`: 让 CVAE 的 posterior 对齐 VAE 目标分布，同时重构 `x`
