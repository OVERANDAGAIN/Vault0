# 六、奖励函数的可计算实现（Computational Realization of Reward）

前文定义了奖励的结构与设计依据，本节<span style="background:#CDF469">进一步</span>说明每一项如何从对话<span style="background:#CDF469">数据中实际</span>计算得到。<span style="background:#CDF469">核心思</span>路是：

<span style="background:#FFB78B">> 将不可直接观测的奖励项，转化为可通过模型或规则估计的信号</span>。

<span style="background:#FFB78B">整体实现框架如下表所示：</span>

| 奖励                        | 类别    | 符号    | 含义                | 类型       | 输出      | 计算方式                  | 数据来源         |
| ------------------------- | ----- | ----- | ----------------- | -------- | ------- | --------------------- | ------------ |
| $r_t^{\text{user}}$       | 即时+长期 | $s_t$ | task satisfaction | 用户任务价值   | $[0,1]$ | BERT/LLM judge        | 人工/ LLM      |
| ^                         | ^     | $c_t$ | continuation      | 留存 proxy | 概率      | 分类模型（continue / stop） | 规则 / 模拟用户    |
| $r_t^{\text{biz}}$        | 商业价值  | $a_t$ | ad acceptance     | 商业收益     | $[0,1]$ | 分类/回归模型（兴趣判别）         | LLM标注 / 模拟用户 |
| $r_t^{\text{constraint}}$ | 约束    | $q_t$ | annoyance         | 负反馈      | $[0,1]$ | 分类模型（负情绪）             | LLM标注 / 模拟用户 |
| ^                         | ^     | $o_t$ | ad overload       | 广告密度     | 标量      | 规则计算（频率）              | 无需训练         |
| ^                         | ^     | $d_t$ | topic deviation   | 话题偏移     | 距离      | embedding 相似度         | 无需标注         |

以下逐项说明具体实现方式。

---

## 6.1 任务完成度（$s_t$）

### 输入

$$(h_t, y_t, u_{t+1})$$

其中：
- $h_t$：对话历史
- $y_t$：当前系统回复
- $u_{t+1}$：用户下一轮反馈

> 必须包含 $u_{t+1}$，否则无法判断回复是否真的对用户有帮助。

### 输出

$$s_t \in [0,1]$$

### 模型实现

**方法 A：监督模型（推荐）**

```
输入：[CLS] history [SEP] response [SEP] next_user
输出：satisfaction score
```

使用 BERT / DeBERTa encoder 后接 MLP 输出标量。

**方法 B：LLM 打标签（更现实）**

Prompt 示例：
> Given the dialogue context and response, does the response help the user solve their problem? Answer with a score from 0 to 1.

再用这些标签训练轻量级模型（知识蒸馏）。

### 数据来源

- 人工标注（少量高质量样本）
- LLM 自动标注（主流做法，可规模化）

---

## 6.2 对话持续性（$c_t$）

### 输入

$$(h_t, y_t)$$

### 输出

$$c_t = P(\text{continue})$$

### 实现方式

**方法 A：规则（最简单）**
- 用户继续说话 → 1
- 用户结束对话 → 0

**方法 B：预测模型（更优）**
- 输入：对话历史 + 系统回复
- 输出：继续对话的概率
- 类似推荐系统中的 retention model

### 数据来源

- 行为规则（显式结束信号）
- 模拟用户交互

---

## 6.3 广告接受度（$a_t$）

### 输入

$$(h_t, y_t, u_{t+1}, z_t)$$

### 为什么需要 $z_t$（topic segment）

广告是否“有效”必须结合当前话题判断：
- 在“论文写作”话题下推荐写作工具 → 合理
- 在“旅游规划”话题下推荐写作工具 → 无关

### 输出

$$a_t \in [0,1]$$

### 模型实现

分类任务：
```
标签：interested / neutral / not interested
```
或回归任务直接输出兴趣分数。

### 数据来源

- LLM 标注（最现实）
- 模拟用户行为

---

## 6.4 用户反感度（$q_t$）

### 输入

$$(h_t, y_t, u_{t+1})$$

### 输出

$$q_t \in [0,1]$$

### 模型实现

情绪/意图分类：
```
标签：reject / annoyed / neutral
```
使用 BERT classifier 或 LLM judge。

### 关键区分

必须区分：
- “不感兴趣”（neutral，可接受）
- “反感”（annoyed，需惩罚）

否则 reward 会产生误导。

### 数据来源

- LLM 标注
- 模拟用户反馈

---

## 6.5 广告密度惩罚（$o_t$）

### 输入

历史广告序列：
$$\{\mathbf{1}[\text{ad}_{t-k}], \dots, \mathbf{1}[\text{ad}_t]\}$$

### 实现方式（规则，无需训练）

**频率惩罚：**
$$o_t = \frac{\text{\#ads in last }k\text{ turns}}{k}$$

**连续广告惩罚：**
```
连续广告长度 > threshold → 额外 penalty
```

> 这一项是工程规则，不需要模型。

---

## 6.6 话题偏移（$d_t$）

这是 **topic segment 真正进入 reward 计算的关键位置**。

### 输入

$$y_t, \quad z_t$$

### 如何得到 $z_t$

$$z_t = \text{avg\_embedding}(\text{utterances in current segment})$$

### 计算方式

$$d_t = 1 - \cos(\text{embed}(y_t), z_t)$$

### embedding 选择

- Sentence-BERT
- OpenAI embedding
- 或任意语义编码器

### 解释

- 高相似度（$\cos \approx 1$）→ 话题一致，$d_t \approx 0$
- 低相似度（$\cos \approx 0$）→ 话题偏移，$d_t \approx 1$

---

## 6.7 整体实现流程总结

> Each reward component is not directly observed but estimated from dialogue context, system responses, and user feedback. Specifically, semantic-based models (e.g., BERT-style encoders or LLM-based judges) are used to estimate user satisfaction, continuation, ad acceptance, and annoyance from textual signals, while rule-based mechanisms are used for exposure control. Topic segments are incorporated as semantic references when computing ad relevance and topic deviation, ensuring that reward estimation is conditioned on the current dialogue context rather than isolated turns.

## 6.8 训练数据说明

为避免 reviewer 追问“模型从哪学”，我们明确说明：

> We train these reward estimators using a combination of LLM-generated annotations and simulated user interactions.

## 6.9 小结

本节完成了从“reward 定义”到“reward 计算”的映射：

- **四项**（$s_t, c_t, a_t, q_t$）通过语义模型估计
- **两项**（$o_t, d_t$）通过规则或相似度计算
- **topic segment** 作为语义条件参与 $a_t$ 和 $d_t$ 的计算

这使得奖励函数不仅在理论上合理，而且在工程上可复现、可落地。
