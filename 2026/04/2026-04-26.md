# Metrics 设计：从 reward-independent evaluation 出发

目前我对 metrics 的基本原则是：**评价指标尽量不依赖我们自己定义的 reward 子项**。
因为如果用 (s_t, q_t, d_t) 这些 reward 内部变量来评价 planner，就会变成“用自己的 reward 评价自己的方法”，说服力不够。

所以更合理的是从可观测 episode log 出发：

$$\mathcal{E} = \{(y_t, u_t, ad_t, adID_t, click_t, topic_t, done_t)\}_{t=1}^{T}$$

也就是只用系统回复、用户回复、是否插广告、广告 ID、是否点击/接受、topic id 和退出信息来定义 metrics。

---

# 当前暂定的四类指标

## 1. Retention：用户是否留下来

最直接的定义是：

$$\text{Retention} = \frac{T}{T_{\max}}$$

其中 $T$ 是实际对话轮数，$T_{\max}$ 是最大轮数。

这个指标的好处是简单、独立、可观测。
但它也有明显问题：**长对话不一定是好对话**。用户可能是因为问题没解决才继续聊，也可能是被系统拖住。

所以这里需要讨论：

> Retention 应该只用 session length 吗？还是要区分“有效留存”和“低效拖延”？


### Retention 指标设计：Session Length 是否足够？

在 CAM（Conversational Ad-Mixing）中，最直观的 retention 指标是：

$$\text{Retention} = \frac{T}{T_{\max}}$$

其中：

* $T$：实际对话轮数（session length）
* $T_{\max}$：预设最大轮数

问题在于：

> **更长的对话是否真的代表更好的用户留存？**


---

### 一、Session Length / Dwell Time 是有价值信号，但不是满意度本身

在信息检索领域，dwell time（停留时间）长期被视为用户满意度的重要 proxy，但研究指出：

> dwell time 是 useful but noisy（有用但有噪声）的隐式反馈，而非满意度的直接等价物。

---
[1] Kim Y, Hassan A, White R W, et al. Modeling dwell time to predict click-level satisfaction[C]//Proceedings of the 7th ACM international conference on Web search and data mining. 2014: 193-202.

用户停留时间确实与 satisfaction 有相关性，但这种关系受到多种上下文影响，例如：

* 页面长度
* 查询意图
* 内容复杂度
* 用户任务类型

因此：

* 停留很久可能是内容优质；
* 也可能是内容难懂、任务复杂、用户迷失。

---


 长对话可能意味着：
 
 （1）有效留存 用户愿意持续互动，因为内容有帮助。
 
（2）低效拖延 用户不断纠偏、系统不断偏题、任务迟迟未解决。


#### 1. 两类信号

一次 episode 可观测为：

$$\mathcal{E} = \{(y_t, u_t, ad_t, click_t, topic_t, done_t)\}_{t=1}^{T}$$

| 变量         | 含义          |
| ---------- | ----------- |
| $T$        | 对话轮数        |
| $y_t$      | 第 $t$ 轮系统回复 |
| $u_t$      | 第 $t$ 轮用户回复 |
| $ad_t$     | 是否插广告       |
| $adID_t$   | 广告 ID       |
| $done_t$   | 是否退出        |
| $t_{exit}$ | 退出轮数        |
些是 **observable signals**：


$$O_t = \{T, done_t, ad_t, click_t, topic_t, u_t\}$$

但用户满意、反感、疲劳等不是直接观测量，而是 **latent  states**：

只能估计：

$$\hat{Z}_t = f_\theta(\mathcal{E}_{1:t})$$

---

#### 2. Reward 可以使用 latent estimator

Reward 是优化目标，本身允许引入估计量：

这些可以由：

* LLM judge
* BERT classifier
* human-labeled reward model

估计。

---


#### 3. Metrics 使用 observable signals

Metrics 用于方法独立评价，因此主指标应尽量来自可观测行为：

$$M_{\text{primary}} = g(\mathcal{E})$$

例如：

$$\text{Retention} = \frac{T}{T_{\max}}$$

$$\text{CAR} = \frac{\sum_t click_t}{\sum_t ad_t}$$

$$\text{AdLoad} = \frac{\sum_t ad_t}{T}$$

$$\text{TSR} = \frac{\sum_{t=2}^{T}\mathbb{I}(topic_t \neq topic_{t-1})}{T-1}$$


---

#### 4. 如果有真人实验，latent states 可通过 survey 观测

真人实验中，可以通过 Likert scale 近似观测 latent variables：

$$score \in \{1,2,3,4,5\}$$
如：
* 任务是否被解决？
* 广告是否自然？
* 是否愿意继续使用？
* 是否感到被打扰？



---

## 2. Monetization：商业收益效率

广告收益可以定义为：

$$\text{Monetization} = \frac{\sum_{t=1}^{T} click_t \cdot v(adID_t)}{T}$$

即每轮对话的平均广告收益


>问题：在我们当前模拟环境里，$click_t$ 并不是真实点击，而是 user 的接受事件。而不是工业广告里的真实 CTR。



---

## 3. Interaction Cost：用户交互成本

广告不只是商业曝光问题，还会改变对话轨迹。因此需要一个成本指标。

暂时可以定义：

$$\text{InteractionCost} = \alpha \cdot \text{Load} + \beta \cdot \text{TSR} + \gamma \cdot \text{CR}$$

其中：

$$\text{Load} = \frac{\sum_t ad_t}{T}$$

表示广告密度。

$$\text{TSR} = \frac{\sum_{t=2}^{T} \mathbb{I}(topic_t \neq topic_{t-1})}{T-1}$$

表示 topic switch rate。

$$\text{CR} = \frac{\sum_t corr_t}{T}$$

表示用户纠偏率，例如用户说“我问的是……”“不是这个”“回到刚才的问题”。

这个指标的想法是把广告压力、话题不稳定和用户纠偏统一成一个 user friction index。



所以这里最值得拿出来讨论：

> Interaction Cost 是应该合成一个指标，还是分别报告 Load、Topic Switch、Correction 三个可观测量？

---

# Pareto：作为最终比较方式，而不是单一分数

因为 CAM 本质是多目标问题，所以我倾向于不把所有指标强行压成一个 scalar，而是用 Pareto view。

策略 $\pi$ 的评价向量可以写成：

$$M(\pi) = \big[ \text{Retention},\ \text{Monetization},\ \text{Conversion},\ -\text{InteractionCost} \big]$$

一个策略如果在不降低 retention 的情况下提高 monetization，或者在不降低 monetization 的情况下降低 interaction cost，就可以认为更优。

这里的讨论点是：

> 最终我们是报告一个 weighted score，还是报告 Pareto frontier？

我个人倾向于先报告 Pareto，因为现在各项权重还没有产品或真实数据校准，强行加权会显得任意。
