**Metrics 不能直接依赖我们自定义的 reward。**
如果 metrics 继续使用 `total_reward / discounted_return / avg_s_t / avg_q_t` 这类内部变量，本质上仍然是在用方法自己的目标函数评价自己，缺乏独立性。

> metrics 的输入应尽可能来自可观测交互结果，而不是 reward decomposition。
> 即只使用 episode log、广告事件、用户行为事件与退出信息。

---

# 一、Metrics 的原始输入

一个 episode 可抽象为：

$$\mathcal{E} = \{(y_t, u_t, ad_t, adID_t, click_t, done_t)\}_{t=1}^{T}$$

其中：

| 变量         | 含义            | 来源                        |
| ---------- | ------------- | ------------------------- |
| $T$        | 对话轮数          | episode log               |
| $y_t$      | 第 $t$ 轮系统回复   | agent log                 |
| $u_t$      | 第 $t$ 轮用户回复   | simulator / real user log |
| $ad_t$     | 是否插入广告        | agent action              |
| $adID_t$   | 广告 ID         | agent metadata            |
| $click_t$  | 用户是否点击 / 接受广告 | simulator / user event    |
| $done_t$   | 是否退出          | environment               |
| $t_{exit}$ | 实际退出轮次        | episode log               |

这个定义的关键在于：
**evaluation 基于 interaction outcome，而不是 reward model。**

---

# 二、设计独立 Metrics

## 1. Rule-based metrics

第一层优先使用可以直接从 log 中计算的指标。
这类指标不依赖模型判断，不依赖 reward，也最接近“硬约束”。

---

### 1.1 对话留存类：Conversation Retention

这一类主要回答：

> 用户是否持续对话？是否被策略提前劝退？

#### 1.1.1 Session Length

$$\text{SessionLength} = T$$

表示单个 episode 的实际对话长度。

若设最大轮数 $T_{\max}$，则可定义：

$$\text{NormalizedRetention} = \frac{T}{T_{\max}}$$

用于消除不同实验配置下最大轮数不同带来的影响。

---

#### 1.1.2 Early Exit

$$\text{EarlyExit} = \mathbb{I}(T < T_{\max})$$

多 episode 下：

$$\text{EarlyExitRate} = \frac{1}{N} \sum_{i=1}^{N} \mathbb{I}(T_i < T_{\max})$$

用于判断用户是否提前结束，而不直接假设退出原因。

---

#### 1.1.3 Exit Turn

$$t_{exit} = T$$

用于分析退出发生在对话前期、中期还是后期，辅助判断广告插入位置的长期影响。

---

### 1.2 广告曝光与收益类：Ad Delivery / Monetization

---

#### 1.2.1 Revenue

若每个广告对应价值函数 $v(adID_t)$：

$$\text{Revenue} = \sum_{t=1}^{T} \mathbb{I}(click_t=1) \cdot v(adID_t)$$

---

#### 1.2.2 Revenue Density

$$\text{RevenueDensity} = \frac{\text{Revenue}}{T}$$

相比总收益，Revenue Density 更适合对话场景，因为它避免“仅靠延长对话长度提高收益”。

---

### 1.3 话题轨迹稳定性：Topic Trajectory Metrics

这一类只依赖 topic metadata，而不直接依赖语义 reward。

---

#### 1.3.1 Topic Switch Count

$$N_{switch} = \sum_{t=2}^{T} \mathbb{I}(topic_t \neq topic_{t-1})$$

---

#### 1.3.2 Topic Switch Rate

$$\text{TopicSwitchRate} = \frac{N_{switch}}{T-1}$$

该指标过高通常意味着 planner 不稳定，存在较强 topic oscillation，可能说明系统为了插广告频繁改变话题路径。

---

## 2. Behavioral metrics

第二层不再只看结构，而是看用户行为反馈。
相比 rule-based metrics，这一层更接近真实用户体验，但解释上仍需谨慎。

例如：

* click / accept
* continue / stop
* correction
* complaint
* repeated clarification
* topic pull-back

需要注意的是：

> 用户继续对话，不一定代表满意；
> 用户未点击广告，也不一定代表反感。

因此 behavioral metrics 更像“行为 proxy”，而不是直接的 latent satisfaction。

这一层可由以下方式估计：

* classifier（BERT / small LM）
* LLM judge
* reward model
* human annotation

---

### 2.1 用户显式摩擦：User Friction

相比直接估计 annoyance，更稳妥的第一步是先抓显式摩擦信号。

---

#### 2.1.1 Correction Count

用户主动将话题拉回原目标，例如：

* “我问的是……”
* “不是这个”
* “回到……”
* “I was asking about...”
* “Actually, I want...”

定义：

$$N_{corr} = \sum_{t=1}^{T} \mathbb{I}(u_t \text{ contains correction})$$

这通常反映系统偏离用户目标后的显式修正。

---

#### 2.1.2 Complaint Count

用户明确表达负反馈，例如：

* “不要广告”
* “别推销”
* “无关”
* “annoying”
* “not related”

定义：

$$N_{complaint} = \sum_{t=1}^{T} \mathbb{I}(u_t \text{ contains complaint})$$

相比 latent annoyance，这类显式 complaint 更容易稳定识别，也更适合作为 early-stage metric。

---

## 3. Judge-based metrics（辅助层）

有些关键维度无法仅通过行为直接观测，例如：

* 广告是否自然
* 是否破坏原任务
* 当前回复是否推进用户目标
* segment coherence

这类指标可以通过：

* LLM judge
* pairwise evaluator
* external classifier

进行估计。

但更合适的定位是：

> Judge-based metrics 用于 diagnostic，而不是 primary evaluation。

否则 metrics 会重新退化为 reward proxy。

---

## 4. Human calibration

完全客观的评价在开放式对话中通常不存在，因此最终需要小规模人工校准。

如果有真人实验，可通过 survey / Likert scale 对 latent states 做近似观测：

$$\text{score} \in \{1,2,3,4,5\}$$

例如：

* 任务是否被解决？
* 广告是否自然？
* 是否愿意继续使用？
* 是否感到被打扰？

Human calibration 的作用不是替代前述指标，而是用于：

* 校准 rule precision
* 检查 behavioral proxy 是否偏移
* 控制 LLM judge bias

---

## 核心上，这里的重点不是构造“另一套 reward”，而是建立一个 evidence hierarchy：

### Rule-based

最稳定、最可复现，但语义有限

### Behavioral

更接近真实用户反馈，但存在解释歧义

### Judge-based

覆盖语义质量，但主观性更高

### Human calibration

作为最终校准层，控制整体偏差

---

$$\text{Rule} \rightarrow \text{Behavioral} \rightarrow \text{Judge} \rightarrow \text{Human Calibration}$$

这样做的目标不是追求绝对客观，而是在可扩展性、独立性与语义覆盖之间取得相对更稳的平衡。
