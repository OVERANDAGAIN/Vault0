
## 1. 这篇论文最核心的贡献到底是什么？

可以这样回答：

这篇论文最核心的贡献不是提出了一个更强的 world model，而是**重新定义了 world model 的用途**。传统 model-based RL 里，world model 主要用于 planning 或 value estimation；但 RWM 把 world model 的预测结果直接当作 **reference trajectory**，然后让 adaptive controller 去追踪这个 reference。也就是说，它把问题从“重新学习一个适应环境的新策略”，转化成了“让真实轨迹重新贴近模型预测轨迹”。论文也明确说，RWM 是把 world model predictions 作为 dynamic control targets，用于 rapid adaptation。

---

## 2. 为什么说 RWM 是“让现实追踪模型”，而不是“让模型追踪现实”？

可以这样回答：

传统 world model 的训练目标是让模型更准确地预测真实环境，也就是 model tracks reality。但 RWM 中，world model 在在线适应阶段被固定下来，它的预测被当作期望状态。此时 adaptive controller 更新的不是 world model，而是 action correction，让真实系统的下一状态靠近预测状态。所以它是 reality tracks model。论文中也写到，forward model 和 controller 使用同一个 error，但优化方向相反：forward model 是让 prediction match observation，而 control module 是让 observation match prediction。

---

## 3. RWM 为什么不需要 reward 也能适应？

可以这样回答：

因为 RWM 的在线适应目标不是直接最大化 reward，而是最小化 latent prediction error。具体来说，world model 预测下一步 latent state，系统执行后得到真实 latent state，二者差距就是 control error。controller 根据这个误差生成 corrective action。只要 world model 的预测代表了原策略在正常动力学下的“理想行为”，那么追踪这个预测就可以恢复原本的 motor behavior。因此在线阶段不需要 reward，只需要 prediction error。论文算法中也是用 (e_t=z_{t+1}-\hat z_{t+1}) 来更新 (\pi_c)，而不是使用 reward。

---

## 4. Base Policy π₀ 和 Adaptive Controller πc 分别负责什么？

可以这样回答：

π₀ 是慢系统，负责长期 reward maximization，也就是决定 agent 应该执行什么行为；πc 是快系统，负责在线纠正 dynamics shift 带来的执行误差。可以理解为：π₀ 决定“做什么”，πc 决定“怎么稳定做到”。论文中明确把 motor learning 分成两类过程：policy learning 逐步发现高价值行为，而 rapid adaptation 在不修改底层 policy 的情况下维持性能。

---

## 5. 为什么只更新 πc，而不更新 π₀ 或 world model？

可以这样回答：

主要是为了实现快速、低成本的 online adaptation。如果 dynamics shift 一出现就更新整个 policy 或 world model，计算量会很大，而且容易破坏原有策略。RWM 的设计是固定 π₀ 和 world model，只在线更新轻量级 controller πc。这样做的好处是保留原始策略学到的 motor program，同时只对执行偏差做局部纠正。论文中也报告，TD-MPC2 full training 约 16.5 小时，而 RWM online adaptation 约 1.8 小时，说明这种设计显著降低了在线适应成本。

---

## 6. RWM 和普通 fine-tuning 的区别是什么？

可以这样回答：

fine-tuning 是 reward-driven 的，它会继续根据 reward 修改策略，因此可能改变原来的 motor program；RWM 是 error-driven 的，它不直接优化 reward，而是保持轨迹接近 world model prediction。这个区别在 Humanoid 实验中特别明显：fine-tuning 虽然可以恢复部分 reward，但 control error 反而上升，说明它可能偏离原有协调模式；RWM 则降低 control error，更好地保留了原 motor structure。论文在 Walker 上也显示，RWM 的 reward recovery 是 94.3%，高于 fine-tuning 的 66.3%。

---

## 7. 为什么说 RWM 更适合真实部署？

可以这样回答：

真实部署中有两个问题：第一，reward 往往不可得、延迟或者很难定义；第二，动力学变化可能是持续的，比如磨损、温度变化或执行器漂移。RWM 不依赖 reward，而是用 prediction error 做快速纠偏，这更符合部署条件。论文在 nonstationary perturbation 中模拟了持续变化的 actuator gain，结果 RWM 的 reward 是 360.56，高于 TD-MPC2 的 311.67 和 no adaptation 的 233.42，同时 control error 更低。

---

## 8. 如果助教问：RWM 的理论保证具体保证了什么？

可以这样回答：

它没有直接保证全局 reward 最优，而是保证：在一定假设下，control error 有上界，并且这个 error bound 可以进一步限制 value loss。也就是说，理论逻辑是：如果系统能持续追踪 world model reference，那么 trajectory deviation 会被控制，最终 performance degradation 也会被控制。这个理论依赖几个条件，包括 world model accuracy、control authority 和 bounded perturbation。论文中的 Theorem 4.2 给出 control error bound，Theorem 4.3 把这个 error bound 连接到 value bound。

---

## 9. 如果 world model 预测错了怎么办？

可以这样回答：

这是 RWM 最重要的局限。因为 RWM 把 world model prediction 当作 reference，如果 world model 在 OOD dynamics 下预测本身错误，那么 controller 可能会高效地追踪错误 reference。所以我在 PPT 里总结为 **Bad model → Bad reference**。不过论文的理论也正好说明了这一点：performance bound 取决于 model accuracy (\epsilon)，如果 (\epsilon) 变大，误差上界和 value loss 都会变差。

---

## 10. 为什么要在 latent space 里做 adaptation，而不是直接在 state space？

可以这样回答：

主要有两个原因。第一，latent space 是由 world model 和 RL agent 学到的，理论上更包含任务相关信息，可以过滤掉无关观测噪声。第二，连续归一化的 latent space 更适合计算 MSE prediction error，也更适合梯度更新 controller。论文中也强调，normalized latent space 能让不同维度对误差计算的贡献更平衡，并且连续 latent space 支持 gradient-based adaptive controller。

---

## 11. 为什么 RWM 不是 Model Predictive Control？

可以这样回答：

MPC 通常会在每一步规划一个未来 action sequence，需要 rollout 或优化一个 horizon；RWM 不做这种在线规划，而是使用 world model 的 single-step prediction 作为即时 reference，然后通过 controller 快速纠偏。因此它更像一个 reflexive controller，而不是 planner。论文讨论部分也指出，RWM 不同于 MPC：它使用 single-step predictions as instantaneous references，因此在线开销更低。

---

## 12. RWM 和 domain randomization 有什么区别？

可以这样回答：

domain randomization 是在训练阶段把很多扰动加入训练分布，希望策略学到更强鲁棒性；RWM 是部署阶段的 online adaptation。两者不是同一种机制。论文也做了 domain-randomized baseline，对 baseline 预训练时加入扰动，但在 evaluation 的 active perturbation 阶段，RWM 仍然取得更高 reward 和更低 control error。这说明 domain randomization 可以增强鲁棒性，但不能完全替代在线适应机制。

---

## 13. 论文实验为什么主要用 actuator perturbation？这会不会限制结论？

可以这样回答：

会，这是论文的局限之一。作者主要测试的是 action space 中的 actuator gain perturbation，包括突发和连续变化。这很适合模拟电机误差、执行器漂移等真实机器人问题，但它还不能证明 RWM 对所有 OOD shift 都有效，比如 morphology change、传感器噪声、地形变化等。论文 future work 也提到，希望扩展到 morphological changes、environmental shifts 和 sensor noise。所以我的判断是：论文已经证明了 RWM 在 actuator-level dynamics shift 下有效，但泛化到更广泛真实环境仍需要进一步验证。

---

## 14. 如果问：RWM 为什么能保持 motor structure？

可以这样回答：

因为 RWM 的目标不是直接追求某个单一 reward，而是追踪 world model 在 latent space 中预测出的整体轨迹。这个 latent trajectory 实际上编码了原策略学到的协调动作模式。特别是在 Humanoid 这种高维系统中，reward 可能只反映前进速度等窄指标，而 control error 更能反映整体 motor program 是否被保持。论文也指出，fine-tuning 可能提高 reward 但破坏 coordination，而 RWM 通过降低 control error 保持 trajectory integrity。

---

## 15. 如果助教问你 PPT 里的“learned cerebellum”是什么意思？

可以这样回答：

这是我的类比，不是论文的正式术语。我的意思是，RWM 的结构和生物运动控制中的小脑功能有相似性：高层 policy 决定目标行为，低层 controller 根据预测误差做快速修正。论文中也提到，这种 separation of concerns 类似于 voluntary movement 和 reflexive correction 的分离，可以用于理解 biological motor control。所以我用 learned cerebellum 来概括它：RL 负责决策，controller 负责反射式纠错。

---

## 16. 如果问：这篇论文最可能被质疑的点是什么？

可以这样回答：

我认为有三个。第一，world model fidelity 是核心瓶颈，如果模型预测不准，reference 就不可靠。第二，目前实验主要集中在 actuator perturbation，还没有充分证明对更复杂 OOD dynamics 的泛化。第三，方法依赖连续可微 latent dynamics 和 policy，这限制了它对离散 action 或不可微系统的直接应用。不过这些质疑并不否定贡献，因为论文主要贡献是提出一个新的范式：把 world model 从 planning module 转化为 reference system。

---

## 17. 如果助教问：你觉得这篇论文和普通 residual policy learning 像不像？

可以这样回答：

表面上像，因为它们都有 base policy 加 correction 的结构。但关键区别在于 correction 的训练信号不同。Residual policy learning 通常仍然围绕 reward 或 task objective 学 residual，而 RWM 的 residual，也就是 (\pi_c)，是由 world model prediction error 驱动的。换句话说，RWM 的 correction 是 reference-tracking based，而不是 reward-optimization based。因此它更接近 adaptive control。

---

## 18. 如果问：为什么不用 KL divergence，而用 MSE？

可以这样回答：

论文里提到，他们采用 MSE 是因为 latent space 被 normalized，不同维度尺度比较一致，所以 MSE 能比较均衡地衡量各维度 prediction error。作者还提到 KL-divergence 这类 scale-invariant metric 在实验中 adaptation 表现更差，因为它可能忽略某些对精细 motor adaptation 很关键的 latent dimensions。

---

## 19. 如果问：RWM 是否改变了原策略 π₀？

可以这样回答：

在线适应时不改变 π₀。π₀ 是预训练好的 base policy，RWM 在线只更新 controller πc。这样做的目的就是保持原策略学到的行为结构，同时只对执行层面的偏差做快速纠正。论文中也强调，rapid adaptation 是在不修改 underlying policy 的情况下维持性能。
