# reward设计： [[reward设计3]]


# formulation具体：

```text
我们将 REC-DIAL 形式化为一个部分可观测的序列决策问题。
每个 episode 是 agent 与 persona-based mock user 之间的一次多轮对话。
在每个 agent decision turn，Planner 接收由 Monitor 构造的压缩 belief state，并输出一个结构化高层动作。
Speaker 将该动作实现为自然语言回复。
随后 Mock User 产生下一轮用户反馈。
用户反馈本身不是 reward，而是 raw feedback；Monitor 和 RewardCalculator 将其解释为下一轮状态信号和当前轮标量 reward。
```
