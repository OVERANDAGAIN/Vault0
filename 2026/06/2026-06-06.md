
---

# 1. 原始 observation $o_t$

在第 $t$ 个 agent decision turn，系统已经看到当前用户输入 $u_t$，并拥有截至当前的完整对话历史：

$$H_t = (u_0, y_0, u_1, y_1, \dots, y_{t-1}, u_t)$$

如果把 previous planner action、广告展示记录、广告点击/拒绝等事件也记录在 episode log 里，那么可以把它们视为 $H_t$ 的一部分。于是原始 observation 定义为：

$$o_t = (p, H_t)$$

其中：

| 符号    | 含义                   |
| ----- | -------------------- |
| $p$   | 用户画像或 persona prior  |
| $H_t$ | 截至当前用户输入的完整对话历史与事件日志 |

这里的 $o_t$ 是原始输入，不直接交给 Planner 做决策。

---

# 2. Monitor 的新职责

Monitor 直接把原始 observation 压缩成 Planner state：

$$s_t = M(o_t)$$

也就是：

$$s_t = M(p, H_t)$$

这里 $M$ 是 Monitor。它的职责是：

```text
读取用户画像和完整对话历史；
抽取当前用户需求与偏好；
维护长期 summary；
保留最近 K 轮局部上下文；
统计广告曝光历史；
输出 Planner 可用的压缩状态 s_t。
```

但 Monitor 仍然不做动作选择。它不能输出：

```text
是否插广告；
插哪个广告；
是否换话题；
换到哪个话题。
```

这些仍然是 Planner 的动作。

---

# 3. State $s_t$

Planner 的 state 定义为：

$$s_t = (p, b_t, \sigma_t, \ell_t^K, e_t^{ad})$$

其中：

| 符号         | 含义                           |
| ---------- | ---------------------------- |
| $p$        | 用户画像或 persona prior          |
| $b_t$      | 用户需求与偏好 belief               |
| $\sigma_t$ | rolling summary，长期对话摘要       |
| $\ell_t^K$ | 最近 $K$ 个 user-agent exchange |
| $e_t^{ad}$ | 广告曝光状态                       |

对应 state space：

$$\mathcal{S} = \mathcal{P} \times \mathcal{B} \times \Sigma \times \mathcal{L}^{K} \times \mathcal{E}^{ad}$$


---

## 3.1 User belief $b_t$ 定义


在第 $t$ 个 agent decision turn，user belief 定义为：

$$b_t = (g_t, P_t^+, P_t^-, C_t^{\text{user}})$$

| 符号           | 含义                   | 说明                       |
| ------------ | -------------------- | ------------------------ |
| $g_t$        | user goal            | 用户当前主要目标                 |
| $P_t^+$      | positive preferences | 用户明确表达的正向偏好              |
| $P_t^-$      | negative preferences | 用户明确表达的负向偏好、拒绝或不想要的内容    |
| $C_t^{\text{user}}$ | user constraints     | 用户给出的显式约束，例如预算、地点、时间、类别等 |

这个版本是比较保守和标准的。它对应的是 dialogue system / conversational recommendation 里常见的 belief state 设计：系统维护用户当前目标、约束条件，以及用户接受或拒绝的偏好信息。






---

## 3.2 Dialogue memory

$$\sigma_t, \ell_t^K$$

其中：

$$\ell_t^K = \text{Tail}_K(H_t)$$

$\sigma_t$ 是长期摘要，保存稳定偏好、明确拒绝、已解决问题、未解决问题、不要重复的内容等。
$\ell_t^K$ 是最近 $K$ 轮原文，用于保留局部语境。

---

## 3.3 Ad exposure state $e_t^{ad}$

$$e_t^{ad} = (n_t^{ad},  \delta_t^{ad}, o_t^{ad})$$

其中：

| 符号              | 含义                                                |
| --------------- | ------------------------------------------------- |
| $n_t^{ad}$      | 当前 episode 已展示广告数量                                |
| $\delta_t^{ad}$ | 距离上一次广告的轮数                                        |
| $o_t^{ad}$      | 最近广告结果，例如 clicked / accepted / ignored / rejected |

这里不放 `user_ad_attitude` 这类心理推测，也不放候选广告集合。它只记录广告曝光与广告事件历史。

---

# 4. Planner action $a_t$

Planner 的动作保持最简：

$$a_t = (d_t, j_t, w_t, k_t)$$

其中：

| 符号    | 含义                 |
| ----- | ------------------ |
| $d_t$ | 是否插入广告             |
| $j_t$ | 如果插广告，选择哪个广告       |
| $w_t$ | 是否切换话题             |
| $k_t$ | 如果切换话题，切换到哪个 topic |

动作空间为：

$$\mathcal{A} = \{0,1\} \times (\mathcal{D} \cup \{\varnothing\}) \times \{0,1\} \times (\mathcal{T} \cup \{\varnothing\})$$

Planner policy 为标准 RL 形式：

$$a_t \sim \pi_\theta(a \mid s_t)$$

如果有硬约束，则定义有效动作空间：

$$a_t \in \mathcal{A}^{\text{valid}}(s_t) \subseteq \mathcal{A}$$



---

# 5. Speaker 输入

Planner 输出：

$$a_t = (d_t, j_t, w_t, k_t)$$

Speaker 接收：

$$(s_t, a_t)$$

并生成自然语言回复：

$$y_t = \text{Speaker}(s_t, a_t)$$

---

# 6. Transition

完整一轮 transition 为：

$$o_t = (p, H_t)$$
$$s_t = M(o_t)$$
$$a_t \sim \pi_\theta(a \mid s_t)$$
$$y_t = \text{Speaker}(s_t, a_t)$$
$$u_{t+1} = \text{UserSim}(H_t, y_t)$$
$$r_t = R(s_t, a_t, y_t, u_{t+1})$$
$$H_{t+1} = H_t \cup \{y_t, u_{t+1}\}$$
$$s_{t+1} = M(p, H_{t+1})$$

最终 RL transition 是：

$$(s_t, a_t, r_t, s_{t+1}, \text{done}_t)$$

---

# 7. 最终模块边界

现在边界可以写得很清楚：

| 模块                   | 输入                    | 输出                      | 职责                                |
| -------------------- | --------------------- | ----------------------- | --------------------------------- |
| Monitor $M$          | $o_t = (p, H_t)$         | $s_t$                   | 将原始 observation 压缩成 Planner state |
| Planner $\pi_\theta$ | $s_t$                 | $a_t = (d_t, j_t, w_t, k_t)$ | 选择广告插入和话题管理动作                     |
| Speaker              | $s_t, a_t$             | $y_t$                   | 生成自然语言回复                          |
| Mock User            | $H_t, y_t$             | $u_{t+1}$               | 模拟用户反馈                            |
| RewardCalculator     | $s_t, a_t, y_t, u_{t+1}$ | $r_t$                   | 计算训练 reward                       |

---

# 8. 最终核心公式

可以把当前 formulation 收敛成：

$$o_t = (p, H_t)$$
$$s_t = M(o_t) = M(p, H_t)$$
$$s_t = (p, b_t, \sigma_t, \ell_t^K, e_t^{ad})$$
$$a_t = (d_t, j_t, w_t, k_t)$$
$$a_t \sim \pi_\theta(a \mid s_t)$$
$$y_t = \text{Speaker}(s_t, a_t)$$
$$r_t = R(s_t, a_t, y_t, u_{t+1})$$
$$(s_t, a_t, r_t, s_{t+1}, \text{done}_t)$$

一句话总结：

> Monitor 负责把原始 observation $o_t = (p, H_t)$ 压缩为 Planner state $s_t$；Planner 根据 $s_t$ 选择是否插广告、插哪个广告、是否换话题、换到哪个话题；Speaker 负责具体语言实现；RewardCalculator 根据用户反馈计算 reward；RL 训练对象是 Planner policy $\pi_\theta(a \mid s_t)$。











可以。我们把 $b_t$ 收敛为最小、最规范的 **core user belief**：

$$b_t = (g_t, P_t^+, P_t^-, C_t^{\text{user}})$$

其中 $b_t$ 只表示系统当前对用户需求和偏好的估计，不再包含用户意图 $i_t$、open questions $Q_t$、满意度、疲劳、广告态度等更主观或派生的信号。

---



# 6. 最终 formulation 片段

可以写成下面这个版本：

$$o_t = (p, H_t)$$
$$s_t = M(o_t)$$
$$s_t = (p, b_t, \sigma_t, \ell_t^K, e_t^{ad})$$
$$b_t = (g_t, P_t^+, P_t^-, C_t^{\text{user}})$$
$$a_t = (d_t, j_t, w_t, k_t)$$
$$a_t \sim \pi_\theta(a \mid s_t)$$

其中：

| 符号    | 含义          |
| ----- | ----------- |
| $d_t$ | 是否插入广告      |
| $j_t$ | 插入哪个广告      |
| $w_t$ | 是否切换话题      |
| $k_t$ | 切换到哪个 topic |

一句话总结：

> $b_t = (g_t, P_t^+, P_t^-, C_t^{\text{user}})$ 是 REC-DIAL 中最小化的 core user belief，表示用户当前目标、正向偏好、负向偏好和显式约束。它不包含意图分类、信息缺口、广告态度或满意度等派生信号，从而保持 belief state 的定义清晰、稳定，并更适合后续 RL Planner 的状态建模。
