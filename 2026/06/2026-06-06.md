
# 最终版 Formulation 总结

## 1. RL step 定义

REC-DIAL 以 **agent decision turn** 作为一个 RL step。

在第 $t$ 个 decision turn，系统已经观察到当前用户输入 $u_t$，并拥有截至当前的对话历史：

$$H_t = (u_0, y_0, u_1, y_1, \dots, y_{t-1}, u_t)$$

然后系统构造 Planner state：

$$s_t = \Psi(H_t, p, E_t^{ad})$$

Planner 根据 $s_t$ 选择高层动作：

$$a_t \sim \pi_\theta(a \mid s_t)$$

Speaker 根据 $s_t$ 和 $a_t$ 生成系统回复：

$$y_t = \text{Speaker}(s_t, a_t)$$

Mock User 给出下一轮反馈：

$$u_{t+1} = \text{UserSim}(H_t, y_t)$$

RewardCalculator 计算当前轮 reward：

$$r_t = R(s_t, a_t, y_t, u_{t+1})$$

最终形成 transition：

$$(s_t, a_t, r_t, s_{t+1}, \text{done}_t)$$

---

# 2. State 定义

$s_t$ 不是当前用户原始输入 $u_t$，也不是完整对话历史 $H_t$。
它是系统在第 $t$ 轮基于当前用户输入、历史摘要、用户画像和广告日志构造出的 **压缩 Planner state**。

最终定义为：

$$s_t = (p, b_t, \sigma_t, \ell_t^K, e_t^{ad})$$

其中：

| 符号         | 含义                           |
| ---------- | ---------------------------- |
| $p$        | 用户画像或 persona prior          |
| $b_t$      | 用户需求与偏好 belief               |
| $\sigma_t$ | rolling summary，长期对话摘要       |
| $\ell_t^K$ | 最近 $K$ 个 user-agent exchange |
| $e_t^{ad}$ | 广告曝光状态                       |

对应 state space：

$$\mathcal{S} = \mathcal{P} \times \mathcal{B} \times \Sigma \times \mathcal{L}^{K} \times \mathcal{E}^{ad}$$

---

## 2.1 User belief $b_t$

$$b_t = (i_t, g_t, P_t^+, P_t^-, C_t^{\text{user}}, Q_t)$$

其中：

| 符号           | 含义                 |
| ------------ | ------------------ |
| $i_t$        | 当前用户意图             |
| $g_t$        | 当前用户目标             |
| $P_t^+$      | 正向偏好               |
| $P_t^-$      | 负向偏好或明确拒绝          |
| $C_t^{\text{user}}$ | 用户约束，例如预算、地点、时间、类别 |
| $Q_t$        | 当前仍未解决的关键问题        |

$b_t$ 只描述用户需求和偏好，不包含广告候选、不包含 reward、不包含 Planner 的动作建议。

---

## 2.2 Dialogue memory

对话历史由两部分进入 state：

$$\sigma_t, \ell_t^K$$

其中：

$$\ell_t^K = \text{Tail}_K(H_t)$$

$\sigma_t$ 保存长期有效信息，例如用户稳定偏好、明确拒绝、已解决问题、未解决问题、不要重复的内容等。

$\ell_t^K$ 保存最近几轮原文，用于处理指代、省略和局部上下文。

---

## 2.3 Ad exposure state $e_t^{ad}$

$$e_t^{ad} = (n_t^{ad}, l_t^{ad}, d_t^{ad}, o_t^{ad})$$

其中：

| 符号         | 含义                                                |
| ---------- | ------------------------------------------------- |
| $n_t^{ad}$ | 当前 episode 已展示广告数量                                |
| $l_t^{ad}$ | 最近窗口内广告负担                                         |
| $d_t^{ad}$ | 距离上一次广告的轮数                                        |
| $o_t^{ad}$ | 最近广告结果，例如 clicked / accepted / ignored / rejected |

这里不放候选广告集，也不放 `user_ad_attitude` 这类心理推测。广告候选属于动作空间中的可选广告池，广告态度可以作为 Monitor / Reward 的辅助信号，但不作为 $e_t^{ad}$ 的核心定义。

---

# 3. Action 定义

Planner 的动作定义为：

$$a_t = (d_t, j_t, w_t, k_t)$$

其中：

| 符号    | 含义                 |
| ----- | ------------------ |
| $d_t$ | 是否插入广告             |
| $j_t$ | 如果插广告，选择哪个广告       |
| $w_t$ | 是否切换话题             |
| $k_t$ | 如果切换话题，切换到哪个 topic |

---

## 3.1 广告决策

$$d_t \in \{0, 1\}$$

其中：

$$d_t = 
\begin{cases}
1, & \text{insert an ad} \\
0, & \text{do not insert an ad}
\end{cases}$$

广告选择为：

$$j_t \in \mathcal{D} \cup \{\varnothing\}$$

其中 $\mathcal{D}$ 是广告池。

结构约束为：

$$d_t = 0 \Rightarrow j_t = \varnothing$$
$$d_t = 1 \Rightarrow j_t \in \mathcal{D}$$

也就是说，Planner 自己决定是否插广告，也自己决定插哪个广告。

---

## 3.2 话题决策

$$w_t \in \{0, 1\}$$

其中：

$$w_t = 
\begin{cases}
1, & \text{switch topic} \\
0, & \text{keep current topic}
\end{cases}$$

目标话题为：

$$k_t \in \mathcal{T} \cup \{\varnothing\}$$

其中 $\mathcal{T}$ 是 topic pool。

结构约束为：

$$w_t = 0 \Rightarrow k_t = \varnothing$$
$$w_t = 1 \Rightarrow k_t \in \mathcal{T}$$

---

# 4. Action space

全局动作空间为：

$$\mathcal{A} = \{0,1\} \times (\mathcal{D} \cup \{\varnothing\}) \times \{0,1\} \times (\mathcal{T} \cup \{\varnothing\})$$

即：

$$a_t = (d_t, j_t, w_t, k_t) \in \mathcal{A}$$

但并不是所有组合都合法。因此定义状态相关的有效动作空间：

$$\mathcal{A}^{\text{valid}}(s_t) \subseteq \mathcal{A}$$

例如：

```text
d_t=0 时，j_t 必须为空；
d_t=1 时，j_t 必须是有效广告；
w_t=0 时，k_t 必须为空；
w_t=1 时，k_t 必须是有效 topic；
敏感内容下不能插广告；
同一个广告不能在短时间内重复展示。
```

policy 仍写成标准形式：

$$a_t \sim \pi_\theta(a \mid s_t), \quad a_t \in \mathcal{A}^{\text{valid}}(s_t)$$

或者：

$$\pi_\theta(a \mid s_t) = 0, \quad \forall a \notin \mathcal{A}^{\text{valid}}(s_t)$$

---

# 5. Speaker 输入

Planner 不决定具体怎么说。
Planner 输出结构化动作：

$$a_t = (d_t, j_t, w_t, k_t)$$

Speaker 接收：

$$(s_t, a_t)$$

然后生成自然语言回复：

$$y_t = \text{Speaker}(s_t, a_t)$$

Speaker 负责：

```text
如何自然承接上下文；
如何表达 topic switch；
如何把广告融入回复；
如何披露广告；
如何控制语气和长度；
如何避免生硬插入。
```

因此，“广告插在哪里”“广告怎么说”“是否自然过渡”都属于 Speaker 的 realization 问题，不进入 Planner action。

---

# 6. Monitor 的位置

Monitor 不做动作选择，不训练 Planner，也不替 Planner 判断是否插广告或是否换话题。

Monitor 的作用是从用户回复中抽取轻量 observation signals，用于更新 state 和 reward proxy：

$$m_t = M(u_t, H_t, a_{t-1}, y_{t-1})$$

它可以输出：

```text
用户意图；
偏好更新；
用户对上一轮动作的反馈；
广告事件，如 accepted / clicked / ignored / rejected；
安全信号；
粗粒度 sentiment / engagement。
```

然后 StateBuilder 使用这些信号更新：

$$s_t = (p, b_t, \sigma_t, \ell_t^K, e_t^{ad})$$

RewardCalculator 可以使用 MonitorOutput 作为辅助证据，但最终 RL 优化对象仍然是 Planner policy：

$$\pi_\theta(a \mid s_t)$$

---

# 7. 最终核心版本

可以把当前 formulation 压缩成下面这几行：

$$s_t = (p, b_t, \sigma_t, \ell_t^K, e_t^{ad})$$
$$\mathcal{S} = \mathcal{P} \times \mathcal{B} \times \Sigma \times \mathcal{L}^{K} \times \mathcal{E}^{ad}$$
$$a_t = (d_t, j_t, w_t, k_t)$$
$$\mathcal{A} = \{0,1\} \times (\mathcal{D} \cup \{\varnothing\}) \times \{0,1\} \times (\mathcal{T} \cup \{\varnothing\})$$
$$a_t \sim \pi_\theta(a \mid s_t), \quad a_t \in \mathcal{A}^{\text{valid}}(s_t)$$

其中：

```text
s_t：压缩后的 Planner state；
d_t：是否插广告；
j_t：选择哪个广告；
w_t：是否换话题；
k_t：换到哪个话题；
Speaker：负责最终语言实现。
```

这个版本边界最清楚：**Planner 学的是广告插入和话题管理策略；Speaker 负责语言生成；Monitor 负责轻量状态感知；RewardCalculator 负责把用户反馈转成训练信号。**
