## 2.5 Ad Exposure & Feedback State ($E_t$)

表示当前 episode 中的广告展示历史，以及用户对广告的可观察反馈。

我们定义：

$$E_t = (\mathbb{1}_k^{\mathrm{ad}}, j_k^{\mathrm{ad}}, \mathrm{out}_k^{\mathrm{ad}})$$

其中：

| 符号 | 含义 |
| --- | --- |
| $\mathbb{1}_k^{\mathrm{ad}} \in \{0, 1\}$ | 第 $k$ 步是否展示广告 |
| $j_k^{\mathrm{ad}} \in \mathcal{D} \cup \{\varnothing\}$ | 第 $k$ 步展示的广告 ID；如果没有展示广告，则为 $\varnothing$ |
| $\mathrm{out}_k^{\mathrm{ad}} \in \mathcal{O}^{\mathrm{ad}}$ | 第 $k$ 步广告的用户反馈结果 |

广告结果集合可以定义为：

$$\mathcal{O}^{\mathrm{ad}}
=
\{
\mathrm{none},
\mathrm{clicked},
\mathrm{accepted},
\mathrm{ignored},
\mathrm{rejected},
\mathrm{complained}
\}$$

如果第 $k$ 步没有展示广告，则：

$$z_k^{\mathrm{ad}} = (0, \varnothing, \mathrm{none})$$

如果第 $k$ 步展示了广告 $j$，但用户没有明显回应，则：

$$z_k^{\mathrm{ad}} = (1, j, \mathrm{ignored})$$

最近 $K$ 步广告事件序列定义为：

$$Z_t^{\mathrm{ad}, K}
=
(z_{\tau}^{\mathrm{ad}}, z_{\tau+1}^{\mathrm{ad}}, \dots, z_{t-1}^{\mathrm{ad}}),
\quad
\tau = \max(0, t - K)$$

也就是说，$Z_t^{\mathrm{ad}, K}$ 记录最近 $K$ 个已完成 decision step 中是否展示广告、展示了哪个广告，以及用户对该广告的直接反馈。

广告负担相关统计可以从 $Z_t^{\mathrm{ad}, K}$ 中派生，而不需要作为 $E_t$ 的顶层变量重复列出。例如，最近 $K$ 步的广告展示次数可以写成：

$$L_t^{\mathrm{ad}}
=
\sum_{k=\tau}^{t-1} \mathbb{1}_k^{\mathrm{ad}}$$

距离上一次广告展示的间隔也可以由 $Z_t^{\mathrm{ad}, K}$ 或完整 episode log 计算得到。

除了近期广告事件序列，REC-DIAL 还维护一个广告反馈记忆 $F_t^{\mathrm{ad}}$。它不是对用户内心广告态度的自由推测，而是 Monitor 基于用户文本和行为事件得到的可追溯摘要。

我们定义：

$$F_t^{\mathrm{ad}}
=
(F_t^{\mathrm{ad}, +}, F_t^{\mathrm{ad}, -}, I_t^{\mathrm{ad}}, \mathrm{note}_t^{\mathrm{ad}})$$

其中：

| 符号 | 含义 |
| --- | --- |
| $F_t^{\mathrm{ad}, +}$ | 用户对广告表现出的正向反馈，例如点击、接受、追问或对某类 sponsored content 表现兴趣 |
| $F_t^{\mathrm{ad}, -}$ | 用户对广告表现出的负向反馈，例如拒绝、抱怨、认为广告打断任务或明确不感兴趣 |
| $I_t^{\mathrm{ad}}$ | 用户关于广告的显式指令，例如 no_more_ads、less_ads、no_specific_category |
| $\mathrm{note}_t^{\mathrm{ad}}$ | 简短自然语言说明，用于记录广告反馈的证据和上下文 |

示例：

```json
{
  "recent_ad_events": [
    {
      "ad_shown": true,
      "ad_id": "travel_2",
      "outcome": "ignored"
    },
    {
      "ad_shown": false,
      "ad_id": null,
      "outcome": "none"
    }
  ],
  "ad_feedback_memory": {
    "positive_feedback": [],
    "negative_feedback": [],
    "instruction": "none",
    "note": "用户没有明确拒绝广告，但对上一条 travel ad 没有追问。"
  }
}
```

因此，$E_t$ 的作用不是保存当前可用广告候选，而是保存广告曝光和广告反馈历史。当前广告候选属于后续 action selection、retrieval 或 valid action set 的内容，不作为 Planner state 的核心组成部分。

Ad Exposure & Feedback State 主要服务于三个后续决策：

1. 判断当前是否适合插入广告；
2. 避免广告过密、重复展示或在负反馈后继续强推；
3. 为 RewardCalculator 提供广告负担和用户反馈相关信号。


## 2.5 AdExposureState $(E_t)$

AdExposureState 是 REC-DIAL 相比普通对话推荐系统必须额外建模的部分。这里我们只关注广告相关的结构化事件数据，包括每一步是否展示广告、展示了哪个广告，以及用户对该广告的直接反馈。

对于每一个已经完成的 decision step $k<t$，定义广告事件：

$$z_k^{\mathrm{ad}}
=
(\mathbb{1}_k^{\mathrm{ad}}, j_k^{\mathrm{ad}}, \mathrm{out}_k^{\mathrm{ad}})$$

其中：

| 符号 | 含义 |
| --- | --- |
| $\mathbb{1}_k^{\mathrm{ad}} \in \{0,1\}$ | 第 $k$ 步是否展示广告 |
| $j_k^{\mathrm{ad}} \in \mathcal{D} \cup \{\varnothing\}$ | 第 $k$ 步展示的广告 ID；如果没有展示广告，则为 $\varnothing$ |
| $\mathrm{out}_k^{\mathrm{ad}} \in \mathcal{O}^{\mathrm{ad}}$ | 第 $k$ 步广告的用户反馈结果 |

广告结果集合可以定义为：

$$\mathcal{O}^{\mathrm{ad}}
=
\{
\mathrm{none},
\mathrm{clicked},
\mathrm{accepted},
\mathrm{ignored},
\mathrm{rejected}
\}$$


在第 $t$ 个 decision step，Planner 不能看到当前动作之后才会产生的广告结果，因此 $E_t$ 只包含已经完成的历史广告事件。我们定义：

$$E_t
=
(z_{\tau}^{\mathrm{ad}},z_{\tau+1}^{\mathrm{ad}},\dots,z_{t-1}^{\mathrm{ad}}),
\quad
\tau=\max(0,t-K)$$

其中，$K$ 是 recent ad history window 的长度。
