$$D_t = (m_t, h_t^K)$$

| 符号 | 含义 | 对应文献思路 |
| --- | --- | --- |
| $m_t$ | long-term dialogue memory / rolling summary | 递归摘要、长期记忆、memory update |

这个拆法有依据：recursive summarization 方法会用 previous memory 和 following context 递归生成新的 memory；MemoryBank 类方法维护长期 memory 并持续更新；MemGPT 则强调有限 context window 下需要把长期 memory 和当前 working context 区分开。([arXiv][1])

## 1. $h_t^K$：recent history window

这个最简单。它不是总结，而是**最近几轮原文**。

完整历史定义为：

$$H_t = (q_0, y_0, q_1, y_1, \dots, q_{t-1}, y_{t-1}, q_t)$$

其中 $q_t$ 是当前用户输入，$y_t$ 还没有生成。

令：

$$\tau = \max(0, t - K)$$

则最近窗口定义为：

$$h_t^K = (q_{\tau}, y_{\tau}, q_{\tau+1}, y_{\tau+1}, \dots, q_{t-1}, y_{t-1}, q_t)$$

也就是说，$h_t^K$ 包含：

```text
最近 K 个已完成 user-agent exchange
+
当前用户输入 q_t
```

这个设计对应 LLM 里的 **working context / context window**：保留最近原文，用来处理指代、省略、语气、局部承接关系。MemGPT 这类工作强调 fixed context length 是限制，因此需要管理当前 context 与长期 memory 的关系。([arXiv][2])

## 2. $m_t$：rolling dialogue memory


$$m_t = M_{\mathrm{mem}}(m_{t-1}, \Delta H_t)$$

其中：

$$\Delta H_t = (y_{t-1}, q_t)$$

即：

$$\text{previous memory} + \text{latest interaction} \rightarrow \text{updated memory}$$

| 操作 | 作用 |
| --- | --- |
| $\mathrm{Extract}_{\text{mem}}$ | 从最新交互中抽取长期有用的信息 |
| $\mathrm{Merge}$ | 和上一轮 memory 合并，更新 resolved / pending / avoid |
| $\mathrm{Compress}$ | 删除冗余、过期、局部细节，保持 memory 简短 |




> $m_t$ 表示由 Monitor 维护的长期 dialogue memory，用于记录对话主线、已解决事项、未解决事项；$h_t^K$ 表示最近 $K$ 个 exchange 以及当前用户输入，用于保留局部上下文。前者是压缩记忆，后者是原文窗口。
