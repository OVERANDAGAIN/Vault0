# 1. 项目当前状态

当前版本已经完成：

```text
1. formulation 主流程接入
2. 原 mock_user / 用户画像驱动逻辑保留
3. DialogueEnv 支持标准 transition
4. RewardCalculator 支持 rule-based + 可选 LLM judge
5. Metrics 独立于 reward
6. Runner 支持配置化实验
7. 本地无 LLM API 配置可以完整跑通
```

当前主循环是：

```python
state = env.reset()

while not done:
    action = planner.act(state, env.action_space)
    next_state, reward, done, info = env.step(action)
    planner.update(state, action, reward, next_state, done)
    state = next_state
```

对应 formulation：

```text
s_t -> a_t -> y_t -> q_{t+1} -> H_{t+1} -> s_{t+1}, r_t
```

# 3. 关键模块和 formulation 对应关系

| formulation 部分        | 代码位置                                   | 作用                                  |
| --------------------- | -------------------------------------- | ----------------------------------- |
| (p_u) 用户画像            | `core/profile.py`                      | `UserProfile`                       |
| Mock user 参数          | `core/profile.py`                      | `SimParams`                         |
| (B_t) BeliefState     | `core/state.py`                        | 用户目标、偏好、满意度、任务进展                    |
| (D_t) DialogueContext | `core/state.py`                        | 历史摘要、最近上下文、广告事件                     |
| (s_t=(p_u,B_t,D_t))   | `core/state.py`                        | `PlannerState`                      |
| (a_t) 一维动作            | `core/planner.py`                      | `ActionSpace`, `PlannerAction`      |
| Speaker               | `core/speaker.py`                      | `(state, action) -> AgentUtterance` |
| Mock User             | `core/simulators/profile_simulator.py` | 用户反馈生成                              |
| Transition            | `env/dialogue_env.py`                  | `step(action)`                      |
| Reward                | `reward.py`                            | `RewardCalculator`                  |
| Metrics               | `metrics.py`                           | `MetricsCalculator`                 |
| Episode log           | `core/types.py`                        | `EpisodeLog`, `TrajectoryStep`      |

# 5. 最小本地运行：不需要 LLM API

推荐先跑这个命令：

```bash
PYTHONPATH=src python scripts/run_simulation.py \
  --experiment experiments/baseline.json \
  --num-episodes 1 \
  --planner rule \
  --monitor rule \
  --speaker mock \
  --reward-judge none \
  --user-response corpus \
  --seed 7
```

含义：

```text
planner=rule          使用规则 Planner
monitor=rule          使用规则 Monitor
speaker=mock          使用 mock speaker，不调 LLM 生成系统回复
reward-judge=none     reward 使用 state/rule-based 路径，不调 LLM judge
user-response=corpus  mock user 回复使用 corpus fallback，不调 User LLM
seed=7                固定随机种子
```

这条命令是最稳的 smoke test。

如果不想保存输出：


# 6. 输出结果在哪里

默认保存到：

```text
output/simulations/
```

文件名类似：

```text
simulation_20260622_021234.json
```

输出 JSON 顶层结构：

```json
{
  "run_id": "...",
  "timestamp": "...",
  "config": {},
  "batch_metrics": {},
  "episodes": []
}
```

每个 episode 包含：

```json
{
  "episode_id": 0,
  "profile_uid": "...",
  "strategies": {},
  "sim_params": {},
  "metrics": {},
  "steps": [],
  "exit_reason": "..."
}
```

每个 step 包含：

```json
{
  "turn_index": 0,
  "q_t": "...",
  "state": {},
  "action": {},
  "y_t": "...",
  "q_next": "...",
  "ad_event": {},
  "reward": 0.0,
  "reward_breakdown": {},
  "next_state": {},
  "done": false,
  "exit_reason": null,
  "metadata": {}
}
```

这已经包含标准 RL transition 所需字段：

```text
s_t      = step["state"]
a_t      = step["action"]["action_id"]
r_t      = step["reward"]
s_{t+1}  = step["next_state"]
done_t   = step["done"]
```

# 7. 运行参数清单

主脚本：

```bash
python scripts/run_simulation.py --help
```

当前参数：

```text
--profile PROFILE
--config CONFIG
--experiment EXPERIMENT
--no-save
--num-episodes NUM_EPISODES
--planner {random,rule,llm}
--monitor {rule,llm}
--speaker {llm,mock}
--reward-judge {none,rule,llm}
--user-response {corpus,hybrid,llm}
--seed SEED
```

## 7.1 `--experiment`

指定实验配置文件，例如：

```bash
--experiment experiments/baseline.json
```

已有三个：

```text
experiments/baseline.json
experiments/conservative.json
experiments/aggressive_ad.json
```

## 7.2 `--num-episodes`

跑多少个 episode：

```bash
--num-episodes 10
```

## 7.3 `--planner`

Planner 策略：

```text
random   随机选择 noop / ad / topic
rule     规则策略，适合 baseline
llm      LLM Planner
```

## 7.4 `--monitor`

Monitor 策略：

```text
rule     规则构造 PlannerState
llm      LLM 构造 / 校准 PlannerState
```

## 7.5 `--speaker`

系统回复生成方式：

```text
mock     mock speaker，不调用 LLM
llm      LLM speaker
```

## 7.6 `--reward-judge`

reward semantic signal 来源：

```text
none     不使用 LLM judge，走默认 state/rule reward
rule     等价于 none
llm      使用 LLMRewardJudge
```

注意：即使 `reward-judge=llm`，reward 公式仍然是当前 formulation 的：

```text
task + satisfaction + continuation + raw ad value + terminal
```

只是 task / satisfaction / continuation 的语义信号可由 LLM judge 估计。

## 7.7 `--user-response`

Mock user 回复生成方式：

```text
corpus   corpus fallback，不调用 LLM，最适合本地测试
hybrid   corpus + LLM 混合
llm      UserSimulator 调 LLM 生成用户回复
```

## 7.8 `--seed`

固定随机性：

```bash
--seed 7
```

## 8.2 experiment 配置

以 `experiments/baseline.json` 为例：

```json
{
  "experiment_name": "baseline",
  "description": "基线实验配置",
  "strategies": {
    "monitor_strategy": "llm",
    "planner_strategy": "llm",
    "planner_policy": "llm",
    "speaker_strategy": "mock",
    "reward_judge_strategy": "none"
  },
  "simulator": {
    "initial_patience": 1.0,
    "ad_sensitivity": 0.5,
    "topic_depth_pref": 3,
    "max_turns": 20,
    "response_strategy": "corpus",
    "corpus_ratio": 1.0
  }
}
```

其中最重要的是：

```text
strategies.monitor_strategy
strategies.planner_strategy
strategies.speaker_strategy
strategies.reward_judge_strategy
simulator.response_strategy
simulator.max_turns
simulator.ad_sensitivity
simulator.topic_depth_pref
```

### simulator 参数含义

```text
initial_patience   初始耐心
ad_sensitivity     广告敏感度，越高越容易对广告负面反应
topic_depth_pref   用户偏好的话题深入轮数
max_turns          episode 最大 decision steps
response_strategy  corpus / hybrid / llm
corpus_ratio       hybrid 时 corpus 使用比例
```

# 12. Metrics 当前包括什么

`metrics.py` 当前输出：

```text
TaskSuccessRate
EarlyFailureRate
Revenue
RevenueDensity
AdExposureRate
ParetoVector = (TaskSuccessRate, RevenueDensity)
```

注意：

```text
metrics 不使用 reward；
metrics 不使用 reward_breakdown；
metrics 只使用 EpisodeLog / AdEvent / exit_reason。
```

Revenue 的计算逻辑：

```text
只有 ad_outcome in {clicked, accepted} 才计入 revenue
rejected / ignored 不计入 revenue
但都会计入 AdExposureRate
```

# 13. Reward 当前包括什么

`reward.py` 当前公式：

```text
r_t =
lambda_task * task
+ lambda_sat * satisfaction_delta
+ lambda_cont * continuation
+ lambda_ad * raw_ad_value
+ terminal_reward
```

其中：

```text
task              任务推进
satisfaction      满意度变化
continuation      用户是否继续交互
ad                只要展示广告，就取广告原始 reward_value
terminal_reward   根据 exit_reason 判断
```

默认不调用 LLM。
设置：

```bash
--reward-judge llm
```

才会启用 `LLMRewardJudge`。
