
# 结束异常：对话进入告别阶段后未能自然终止

该片段反映出一个典型的结束异常：用户已经多次表达“当前对话应结束、后续再回来继续”的意图，但系统没有及时触发终止，而是持续生成礼貌性告别回复，使对话在无实际任务推进的状态下继续循环。日志证据来自该对话片段。

## 1. 用户已经出现明确的未来再联系意图，但系统仍判断为应继续对话

### 证据摘录

```text
--- Step 6 ---
Agent: ...到时候如果看到心仪的房源，或者想对比几个选项，随时回来找我，我帮你一起分析。祝你周末探路顺利！
User: 谢谢！周末走完一圈后，如果有任何发现或疑问，我一定回来找你聊聊。期待这次搬家能顺利找到一个步行15分钟内的舒适小窝！祝你也有个愉快的周末！
Reward: 1.550 | {
  'task': 0.6,
  'satisfaction': 0.05,
  'continuation': 0.9,
  'terminal': 0.0,
  'metadata': {
    'judge_signals': {
      'task_progress': 0.6,
      'satisfaction_after': 0.85,
      'continuation': 0.9,
      'task_completed': False,
      'exit_reason': None
    }
  }
}
```

### 说明

这里用户的表达已经不是继续当前任务，而是明确表示“周末走完一圈后再回来聊”。这属于典型的延后继续场景，当前会话已经自然收束。

但 reward judge 仍给出：

```text
continuation = 0.9
task_completed = False
terminal = 0.0
exit_reason = None
```

这说明系统将“以后再回来聊”的表达误判为当前对话仍应继续，而没有识别为当前轮次应自然结束。

## 3. 结束阶段的无效轮次仍然获得较高正奖励

### 证据摘录

```text
--- Step 8 ---
Agent: ...等你带着好消息或问题回来，我们随时接着聊。回头见！
User: 好嘞，那我这就出发了！带着地图和期待去探索那片区域，希望回来能给你带来好消息。回头聊！
Reward: 1.200 | {
  'task': 0.3,
  'satisfaction': 0.0,
  'continuation': 0.9,
  'terminal': 0.0,
  'total': 1.2,
  'metadata': {
    'judge_signals': {
      'task_progress': 0.3,
      'continuation': 0.9,
      'task_completed': False,
      'exit_reason': None
    }
  }
}
```

### 说明

从内容上看，Step 8 已经没有新的推荐、分析或决策。Agent 只是重复“祝你顺利、回来再聊”，User 也只是重复“我出发了、回头聊”。

但这一轮仍然获得：

```text
total = 1.2
continuation = 0.9
```

这会导致一个严重问题：系统会把“继续寒暄”视为正收益行为。也就是说，即使对话已经没有任务推进，policy 仍可能学习到继续生成告别语可以获得 reward。

## 4. 多轮告别之后，task_progress 已经降为 0，但 continuation 仍保持高分

### 证据摘录

```text
--- Step 11 ---
Agent: 好的，祝你周末探索充满惊喜！用心走，用眼观察，那个最适合你的角落一定会被你发现。等你带着故事和照片回来，我们慢慢聊。回见！
User: 好嘞，那我就带着地图和好心情出发啦！周末回来见，希望能带回让你也眼前一亮的好消息！回见！
Reward: 0.900 | {
  'task': 0.0,
  'satisfaction': 0.0,
  'continuation': 0.9,
  'terminal': 0.0,
  'total': 0.9,
  'metadata': {
    'judge_signals': {
      'task_progress': 0.0,
      'continuation': 0.9,
      'task_completed': False,
      'exit_reason': None
    }
  }
}
```

### 说明

这一轮的 `task_progress` 已经是：

```text
task_progress = 0.0
```

这说明系统自身已经判断该轮没有实质任务推进。

但同时仍然给出：

```text
continuation = 0.9
terminal = 0.0
```

这两个信号存在明显冲突：

* 如果任务没有推进，并且用户持续表达结束意图，则 continuation 不应继续维持高值；
* 如果当前轮只是告别寒暄，则更合理的判断应是触发自然结束；
* “无任务推进的继续对话”仍然获得正收益。
