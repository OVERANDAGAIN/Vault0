是的，`current_*` 按语义上应该会随对话变化，但两个字段的变化机制不一样。

最清楚的定义是：

```text
initial_topic / initial_task_intent:
    初始 query 的来源信息，episode 内不变，用来追溯第一句话从哪里来。

current_topic / current_task_intent:
    当前对话状态，可以随着对话更新。
```

## 1. `current_topic` 应该更容易变化

`current_topic` 表示当前正在聊的领域，例如：

```text
Food / Travel / Entertainment / Work & Living / Social / Lifestyle / Education
```

它应该在这些情况下变化：

| 场景                        | current_topic 是否变化 |
| ------------------------- | ------------------ |
| 初始 query 进入对话             | 初始化                |
| planner 选择一个 topic action | 更新为该 topic         |
| 用户明确 switch topic         | 更新                 |
| planner 插入广告              | 不更新                |
| 用户继续当前问题                  | 不更新                |

例如：

```text
初始 query 来自 Food: Diet
=> current_topic = Food

planner 后面选择 Travel topic
=> current_topic = Travel

planner 插入 food_0 广告
=> current_topic 不变，不应该变成 Food
```

最后一点很重要：**广告不能反过来改 current_topic**。否则会出现自证循环：推了 Food 广告，然后把当前话题改成 Food，再算出 topic_match=1。

---

## 2. `current_task_intent` 也可以变化，但应该更保守

`current_task_intent` 表示当前任务类型，例如：

```text
Recommendations
Planning and Logistics
Advice & Recommendations
Creative Content Generation
Problem Solving
```

它不是 `continue / switch_topic / exit` 这种 dialogue intent。

它应该在这些情况下变化：

| 场景                                        | current_task_intent 是否变化 |
| ----------------------------------------- | ------------------------ |
| 初始 query 有 source_category/categories     | 初始化                      |
| 用户提出新的任务需求                                | 更新                       |
| planner 切换到一个带 task intent 的新 topic/query | 可以更新                     |
| 普通继续对话                                    | 不更新                      |
| 插入广告                                      | 不更新                      |
| 用户只是说 continue/switch_topic/exit          | 不能直接作为 task intent       |

所以 `current_task_intent` 不是每一轮都变。多数 episode 里，它可能保持初始任务类型，除非用户或 planner 明确把任务转到另一类。
